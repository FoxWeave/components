Cloudant is a cloud-hosted NoSQL database service (DBaaS). See online docs for more details (and demos).

Configuring the Cloudant output connector is quite simple:

  1. Select the Cloudant Account (press the button to add an account, if needed).
  2. Select the name of the target database (the DB Name field).
  3. After selecting the database name (step #2), specify the structure of the JSON document you would like to create in the target Cloudant Database.
  4. Go to the Data Mapping tab and map the pipeline data into the target document structure (defined in step #3).
Tip: Make sure the JSON document field values (defined in step #3) have human readable values. This makes the mapping process (step #4) a lot easier.
Tip: The Cloudant output connector uses a default batch size of 200K bytes. If you are running your integrations On-Premise, you can increase this batch size to whatever value you wish by setting the CLOUDANT_DEFAULT_MAX_BATCH_SIZE system property (i.e. using -D) e.g. to set it to 1Mb, add the following to the container's startup script:

-DCLOUDANT_DEFAULT_MAX_BATCH_SIZE=1024000
