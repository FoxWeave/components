PostgresSQL is a very popular open source Relational Database. See online docs for more details (and demos).

Configuring the PostgresSQL input connector is quite simple:

  1. Select the PostgresSQL Server (press the button to add an account, if needed).
  2. Enter the name of the Database from which you wish to synchronize data.
  3. Enter the SQL Statement (without adding a where clause).
  4. Select the Sequence Field to be used by FoxWeave to keep track of records that is has already sync'd.
Tip: When creating a PostgresSQL Server (after pressing the button), do NOT enter a local loopback address ("localhost" or "127.0.0.1") as the PostgresSQL Server Host name/IP. Remember that this PostgresSQL Server Host address needs to be resolvable on the FoxWeave target on which you chose to start/run the integration i.e. on an On-Premise or On-PaaS host machine.
Tip: When entering the SQL Statement (step #3), DO NOT include a where clause in the statement. This will be added automatically when you select the Sequence Field (step #4).
Tip: The Sequence Field (selected in step #4) is used by FoxWeave to keep track of records that have already been sync'd. This field need to of a type and value that can be tested (in the generated where clause) for an ascending value e.g. an ascending id value field. FoxWeave uses the value of this field (from the last record sync'd) as part of a generated greater-than where clause e.g.:

select A, B, C from mytable where id > ${lastMessage.id}.
